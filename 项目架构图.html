二、文档入库流程 (Ingestion Pipeline)

这条流程用于把正式案例或标准模板写入 PostgreSQL + Qdrant。用户聊天时上传的会话附件不走这条链路。

graph LR
    A["📤 用户上传正式文档
.txt .md .pdf .doc .docx .xls .xlsx"] --> B["📥 API 接收
POST /api/documents/upload
或 /api/templates/upload"]
    B --> C["🔌 文本提取
pypdf / python-docx / openpyxl / xlrd
.doc 走 Word COM / antiword / OLE 回退"]
    C --> D["🧻 文本规范化
parser.normalize"]
    D --> E["✂️ 段落切分
parser.segment"]
    E --> F["🏷️ 争议焦点标注
parser.dispute_tags"]
    F --> G["🔘 向量化
Gemini Embedding
768 维"]
    G --> H["💾 存入 PostgreSQL
documents + paragraphs"]
    G --> I["📳 存入 Qdrant
legal_documents + legal_paragraphs"]

八、独立类案检索流程

这条图对应独立类案检索接口 /api/similar-cases/compare，不再走聊天接口。

graph TD
    U["👤 用户上传聊天附件并切到类案检索模式"] --> FE["🖥️ 前端 executeSimilarCaseSearch

    FE --> LOAD["📂 similar_case_search._load_uploaded_materials
读取 session 的 chat_attachment"]
    LOAD --> PARSE["🧩 parse_document
规范化 + 段落切分 + 争议标签"]
    PARSE --> PROFILE["🧭 抽取案件检索画像
法律关系 / 争议焦点 / 标的 / 主体 / 时间 / 金额 / 关键事实"]
    PROFILE --> QBUILD["🧪 构建 comparison_query
画像字段 + 用户检索要求"]
    QBUILD --> RET["🔍 dual_retrieve + rank_and_aggregate
文档召回 + 段落召回 + 重排"]
    RET --> SCORE["📐 综合评分
doc向量 + 段落分 + 文本重合 + 画像命中 + 文件名 bonus"]
    SCORE --> CLASSIFY["🧷 结果分类
exact_duplicate / near_duplicate / similar_case"]
    CLASSIFY --> RES["📋 返回结构化结果
case_search_profile + matches + citations"]

    TEMPONLY["⚠️ 不污染正式库
上传附件仅存 session_temp_file_store

    LOAD -.-> TEMPONLY

十、观点预测链路（模板管理 + 预测执行）

观点预测是独立链路：先维护案件模板，再按用户问题启动预测，不复用合同审查模板和普通类案接口。

graph TD
    U["👤 用户在观点预测面板维护案件模板"] --> TM["📡 /api/prediction/templates*
创建 / 查询 / 删除 / 追加资产"]
    TM --> EX["🔌 文本提取与解析
案情材料(必填) + 对方语料(选填)"]
    EX --> PT["💾 写入 prediction_* 表
templates / assets / asset_paragraphs"]

    Q["💬 用户在聊天区提问
例如：对方会从什么角度辩驳?"] --> START["📡 POST /api/opponent-prediction/start
session_id + template_id + query"]
    START --> PROFILE["🧭 build_case_profile
规则候选 + LLM 重构案件画像"]
    PROFILE --> RET["🔍 对方视角检索
基于画像与问题生成检索方向"]
    RET --> ARG["🧠 观点生成与重排
主打 / 次打 / 补充 + 引用支持项"]
    ARG --> COUNTER["🛡️ 我方应对建议生成
并区分推断项与证据项"]
    COUNTER --> SNAP["💾 保存预测快照
prediction_report_snapshots"]
    SNAP --> UI["🖥️ 返回预测报告卡片
标题 / 摘要 / 观点 / 引用"]
