二、对话流程（含记忆 + 闲聊分流）

每次对话：保存用户消息 → 加载历史 → 判断闲聊/法律 → 调用 LLM → 保存助手回复。闲聊不走检索管线。

graph TD
    Q["💬 用户提问"] --> SAVE["💾 保存用户消息
MemoryStore.save_message"]
    SAVE --> CHECK{"_should_skip_retrieval?"}
    CHECK -->|"是（闲聊）"| CASUAL["🎈 闲聊路径
handle_casual_chat
CASUAL_SYSTEM_PROMPT + 历史"]
    CHECK -->|"否（法律问题）"| ATT{"有 chat_attachment?"}
    ATT -->|"是"| FOCUS["🎯 附件聚焦"]
    ATT -->|"否"| RET["🔍 双层检索"]
    FOCUS --> RET2["🔍 双层检索"]
    RET -->|"有证据"| CTX["🧱 构建上下文 + 历史消息"]
    RET -->|"无证据"| CTX2["🧱 仅历史 + non-grounded"]
    RET2 --> CTX
    CTX --> GEN["🤖 DeepSeek 生成回答"]
    CTX2 --> GEN
    CASUAL --> GEN2["🤖 DeepSeek 闲聊回答"]
    GEN --> SAVE2["💾 保存助手回复"]
    GEN2 --> SAVE2
    SAVE2 --> RES["📋 返回 ChatResponse"]

三、Agent Pipeline 自纠正循环

Planner 根据输入复杂度动态生成 ExecutionPlan。Validator 检查输出：失败且可重试 → 重新规划 → 再执行。

graph TD
    INPUT["📥 请求输入"] --> CLASSIFY["🔍 classify_complexity
SIMPLE / MEDIUM / COMPLEX"]
    CLASSIFY --> PLAN["📋 Planner 生成 ExecutionPlan
条件分支 + 可选步骤"]
    PLAN --> FILTER["⚙️ _filter_plan
评估 condition 过滤步骤"]
    FILTER --> EXEC["⚡ Executor 执行
调用真实 service"]
    EXEC --> VALID["✅ Validator 校验
ValidationRule 链"]
    VALID -->|"Pass"| OUT["📤 ValidatedOutput"]
    VALID -->|"Fail (retryable)"| CHECK{"retries < max?"}
    CHECK -->|"是"| REPLAN["🔄 重新规划
Rejection 传回 Planner"]
    REPLAN --> PLAN
    CHECK -->|"否"| FAIL["📤 Rejection"]
    VALID -->|"Fail (non-retryable)"| FAIL

    EXEC -.->|"记录"| TLOG["📊 TrajectoryLogger"]
    VALID -.->|"记录"| TLOG
    PLAN -.->|"记录"| TLOG
