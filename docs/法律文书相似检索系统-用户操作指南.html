法律文书相似检索系统
用户操作指南

这份文档是给日常使用系统的人看的，不是给开发人员看的。 你不需要懂代码，只要照着页面上的按钮一步一步操作，就能完成文件上传、普通检索、合同审查、观点预测和类案检索。

1. 先认识页面

位置 你能做什么
左侧菜单 进入“文件管理”“合同审查”“观点预测”“系统状态”等功能区。
中间聊天区 输入问题、查看回答、查看合同审查结果、观点预测结果和类案检索结果。
输入框左侧按钮 上传文件，或者切换成“合同审查”“观点预测”“类案检索”模式。
右侧栏 查看当前会话附件，或者查看回答引用了哪些文件片段。
记住一个最实用的原则： 不同功能用的“文件”不完全一样。普通聊天和类案检索主要看当前会话附件；合同审查要用标准模板加待审合同；观点预测要先建立案件模板。

2. 开始使用前，你要确认什么

如果系统已经有人帮你启动好了，你只需要打开网页即可。

如果需要你自己打开系统，通常使用下面两个地址：

  • 前端页面：http://localhost:3000
  • 后端服务：http://localhost:8000
如果网页能打开，但一直查不到结果，先点左侧“系统状态”看看 PostgreSQL、Qdrant、Embedding、Reranker 是否已经就绪。

3. 最常用的 4 个功能怎么用

3.1 普通聊天检索：像聊天一样提问

适合什么场景： 你想直接问系统一个法律问题，或者让它基于数据库里的材料给出一个回答。
  1. 直接在输入框输入你的问题。
  2. 点击发送按钮。
  3. 如果你想让回答结合你手头的材料，先上传聊天附件，再提问。
  4. 回答出来后，如果页面里出现“引用来源”，可以在右侧栏查看具体引用了哪几行内容。
说人话的建议： 问题不要只写“帮我看一下”，最好写成“请结合我上传的材料，判断对方是否构成违约，并说明依据”。

3.2 合同审查：先准备模板，再审目标合同

适合什么场景： 你手里有一份待审合同，想按照某个标准模板去比对、找问题。
  1. 先点左侧“合同审查”。
  2. 在左侧模板库上传标准合同模板。模板会显示在“模板列表”里。
  3. 回到聊天区，点击输入框左侧的“合同审查”按钮，让它进入合同审查模式。
  4. 再点击回形针上传待审合同。
  5. 在输入框里写清楚你的要求，例如“请重点检查付款条款、违约责任和解除条件”。
  6. 发送后，系统会先帮你匹配模板，再生成审查结果。
容易混淆的点： 左侧上传的是“标准模板”，聊天区上传的是“待审合同”。这两个不是一回事。

3.3 观点预测：先建案件模板，再问问题

适合什么场景： 你想知道“对方大概率会怎么说”“对方最可能从什么地方攻击我方证据”“我该怎么防”。
  1. 点击左侧“观点预测”。
  2. 填写案件名称。
  3. 上传“案情材料”。这是必填项。
  4. 如果你有答辩状、聊天记录、律师函等对方材料，也可以上传到“对方语料”。这一步是选填，但通常很有帮助。
  5. 点击“保存为案件模板”。
  6. 回到聊天区，点“观点预测”按钮切换模式。
  7. 直接输入问题，例如：
    • “对方最可能从哪些角度辩驳？”
    • “对方最可能攻击我方哪条证据？”
    • “如果对方主打诉讼时效，会怎么说？”
  8. 系统会提示你选择案件模板，然后给出预测报告。
想让结果更像真人会怎么说： 对方语料越接近真实答辩口径，预测出来的话术通常越贴近真实场景。

3.4 类案检索：拿你上传的材料去找相似案件

适合什么场景： 你已经有一份案件材料，想看看库里有没有相似案例、同案不同版本，或者相近争议的案件。
  1. 先在当前会话里上传聊天附件。
  2. 再点击输入框左侧“类案检索”按钮。
  3. 你可以直接发送，也可以补一句要求，例如“优先找物业费拖欠和违约金争议的案件”。
  4. 系统会返回：
    • 是否命中同案
    • 高度相似候选
    • 普通相似案例
    • 每个结果的原因、分数和引用片段
这里最重要的一句： 一定要先上传文件，再在同一个会话里使用“类案检索”。如果当前会话没有聊天附件，这个功能就没有材料可以比对。

4. 文件到底要传到哪里

你要做的事 文件应该传到哪里
普通聊天时，让回答参考你的材料 上传到当前会话附件
做类案检索 上传到当前会话附件
做合同审查的标准参照 上传到左侧“合同审查”里的标准模板库
上传待审合同 在聊天区进入合同审查模式后，用回形针上传
做观点预测 上传到左侧“观点预测”里的案件模板

5. 右侧栏怎么用

附件

这里能看到当前会话里已经上传过哪些文件。类案检索时，系统主要就是用这里的文件来比对。

引用来源

这里能看到系统回答时具体引用了哪个文件、哪一段。适合用来核对系统有没有“胡说”。

6. 常见问题

为什么我上传了文件，但回答里没用上？

先确认你是不是上传到了“当前会话附件”，而不是传到了模板库或别的功能面板。

为什么合同审查没开始？

通常是因为没有标准模板，或者没有上传待审合同。

为什么观点预测没有结果？

通常是因为还没有先建立案件模板，或者没选模板就直接问了问题。

为什么类案检索结果很少？

可能是当前会话没有附件，也可能是你的补充要求太窄。可以先上传更完整的材料，再把问题写得具体但不要过窄。

我看不懂“同案命中 / 高度相似 / 类案”有什么区别。

简单理解： “同案命中”最接近原文； “高度相似”大概率是非常近的版本或非常接近的案件； “类案”则是争议点、事实结构或法律关系相近，但不一定是同一个案子。

7. 推荐的使用顺序

  1. 先看“系统状态”，确认服务正常。
  2. 普通检索或类案检索前，先把材料传到当前会话附件。
  3. 合同审查前，先建好标准模板。
  4. 观点预测前，先建好案件模板。
  5. 结果出来后，养成查看“引用来源”的习惯。
